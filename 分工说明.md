A同学：后端与数据负责人
核心职责：负责整个系统的“骨架”和“大脑”，包括数据存储、业务逻辑处理、API接口提供。

详细任务清单：

第一阶段：项目设计与技术选型 (第1周)

数据库设计：
根据需求，设计数据库表结构（E-R图）。
至少包含：用户表、个人基本信息表、教育经历表、工作经历表等。
可以扩展：联系人表、日程表、备忘录表。
明确字段类型、主键、外键、索引。
技术选型主导：
确定后端技术栈（如：Java + Spring Boot + MyBatis，或 Python + Django/Flask）。
确定数据库（如：MySQL，或轻量级的SQLite）。
API接口设计：
（极其重要！） 与B同学共同定义前后端交互的API规范（RESTful API）。
使用工具（如 Swagger, Postman, 或一个简单的Markdown文档）清晰地写出每个接口的：
请求路径 (/api/user/login)
请求方法 (POST, GET, PUT, DELETE)
请求参数 ({ "username": "string", "password": "string" })
返回数据格式 ({ "code": 200, "message": "success", "data": { ... } })
这个API文档就是前后端之间的“合同”。
第二阶段：核心后端功能开发 (第2-3周)

搭建项目环境：初始化Spring Boot/Python项目，集成数据库连接。
用户认证模块：
实现用户注册、登录（建议使用JWT或Session进行会话管理）。
实现密码加密存储。
提供登录状态验证接口。
个人信息CRUD模块：
实现个人基本信息的增、删、改、查接口。
实现教育经历、工作经历等模块的CRUD接口。
所有接口都要考虑权限校验（即用户只能操作自己的数据）。
第三阶段：高级功能与整合 (第4-5周)

扩展功能API：根据项目进度，实现联系人管理、日程管理等额外模块的API。
文件上传功能：实现头像或附件上传的接口（处理文件存储）。
后端单元测试：对关键业务逻辑编写单元测试，保证代码质量。
部署与环境配置：将项目打包（如生成Jar包），准备部署到服务器。
协助联调：配合B同学进行前后端接口联调，快速定位并修复后端问题。
B同学：前端与用户体验负责人
核心职责：负责整个系统的“面孔”和“交互”，包括页面布局、用户操作、数据展示。

详细任务清单：

第一阶段：项目设计与技术选型 (第1周)

UI/UX设计：
绘制系统主要页面的线框图或原型图（可用Figma, Axure, 甚至纸笔）。
确定整体配色、字体、布局风格，保证界面美观、统一、易用。
规划用户操作流程（如：登录后进入主页，点击菜单项跳转到不同功能区）。
技术选型主导：
确定前端技术栈（如：Vue.js + Element Plus，或 React + Ant Design，或原生HTML/CSS/JS）。
确定前端构建工具（如：Vite, Webpack）。
API接口确认：
与A同学一起审阅并确认API文档，确保理解每个接口的用途和数据格式。
第二阶段：核心前端功能开发 (第2-3周)

搭建项目环境：初始化前端项目（如 npm create vue@latest）。
静态页面开发：
根据UI设计，开发所有页面的HTML结构和CSS样式，包括：登录页、注册页、主页仪表盘、个人信息页等。
此阶段可以先不调用接口，用“假数据”（mock data）填充页面，专注布局和样式。
核心功能逻辑实现：
使用JavaScript（或Vue/React）实现页面的动态效果。
根据API文档，对接后端接口，实现：
用户登录、注册功能。
个人信息的展示、编辑、保存功能。
数据的表单验证。
（关键技巧） 在后端接口未完成时，可以使用Mock.js或拦截请求返回本地JSON，模拟数据，实现前端逻辑的开发。
第三阶段：高级功能与整合 (第4-5周)

扩展功能界面：开发联系人管理、日程管理等功能的用户界面和交互。
用户体验优化：
实现响应式设计，让页面在不同分辨率的设备上都有良好的显示效果。
添加加载动画、操作提示（成功/失败）、删除确认等，提升友好度。
前端联调：与A同学配合，将前端代码与真实的后端API进行联调，解决跨域、数据格式不匹配等问题。
文档与演示准备：
负责制作最终答辩PPT的UI设计部分和系统演示视频录制。
负责撰写项目说明书中关于前端技术实现和功能展示的部分。
协作流程与工具
版本控制 - 必备！

工具：Git + Gitee / GitHub
流程：
创建一个远程仓库，main分支为稳定分支。
A和B同学分别创建自己的功能分支，如 feature/backend-login、feature/frontend-profile。
每人完成一小部分功能后，提交到自己的分支，然后向 main 分支发起合并请求。
另一人进行Code Review（代码审查），确认无误后合并。这能有效避免代码冲突和错误。
沟通机制

每日站会：每天花10-15分钟，快速沟通昨天做了什么、今天计划做什么、遇到了什么问题。这是解决障碍、保持同步的最好方法。
即时通讯：使用微信/QQ群进行日常问题沟通。
项目管理与文档

共享文档：可以使用腾讯文档、石墨文档或Notion，共同编写API文档、会议纪要、需求列表。
任务跟踪：使用简单的To-Do List或Trello看板，跟踪任务进度。
项目里程碑（时间表示例）
第1周末：完成需求分析、数据库设计、API接口文档初稿、UI原型图。
第2周末：完成用户认证模块（后端API + 前端页面），实现登录注册功能。
第3周末：完成核心个人信息CRUD模块（后端API + 前端页面）。
第4周末：完成至少1-2个扩展功能（如联系人管理），并完成第一轮前后端联调。
第5周末：完成所有功能，进行系统测试、Bug修复，开始撰写项目报告和准备答辩。
